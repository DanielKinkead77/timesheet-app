Updated database schema for the timesheet app — four tables

users Already exists — no changes needed
idPKbigintAuto-incrementing unique identifier
namestringFull name
emailstring uniqueLogin email
passwordstringHashed by Laravel automatically
timestampsautocreated_at and updated_at
hourly_rates Lookup table — seeded with default rates
idPKbigintAuto-incrementing unique identifier
rate_codestring uniquee.g. NORMAL, OVERTIME, DOUBLE_TIME
descriptionstringe.g. "Normal Rate", "Overtime", "Double Time"
multiplierdecimal(3,2)e.g. 1.00, 1.50, 2.00 — applied to base rate
timestampsautocreated_at and updated_at
projects One user can have many projects
idPKbigintAuto-incrementing unique identifier
user_idFKbigintLinks to users.id
departmentstringe.g. "Engineering", "Design"
descriptionoptionaltextLonger notes about the project
is_activebooleantrue = active, false = archived
timestampsautocreated_at and updated_at
user_id → users.id — deletes cascade
time_sheets Core table — one entry per work session
idPKbigintAuto-incrementing unique identifier
user_idFKbigintLinks to users.id — who logged this
project_idFKbigintLinks to projects.id — which project
hourly_rate_idFKbigintLinks to hourly_rates.id — normal/OT/double
datedateThe date the work was done
start_timetimeWhen work started e.g. 09:00
end_timeoptionaltimeNull means timer is still running
hour_numberoptionaldecimal(5,2)Calculated from start/end, or entered manually
work_descriptionoptionaltextWhat was worked on
timestampsautocreated_at and updated_at
user_id → users.id, project_id → projects.id, hourly_rate_id → hourly_rates.id
